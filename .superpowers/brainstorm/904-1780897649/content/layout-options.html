3 Homepage Layouts — Terminal + Skills + Projects

Same ingredients, different compositions. The terminal hero is the anchor; how it flows into skills and projects varies.

A. Terminal-First, Scroll-Down

~/portfolio $ whoami
> Jamie | Full-Stack Developer
# 3s typewriter animation
~/portfolio $ ls skills/
> React TypeScript Go Python Java...
~/portfolio $ █

→ Terminal 占 40% 视口，是 Hero
→ 向下滚动进入 Bento Grid：技能云 + 4 精选项目卡片
→ 终端打字完成后自动缩小为顶部状态栏
→ 传统滚动节奏，面试官熟悉

B. Split Screen — Terminal + Bento Side-by-Side

$ whoami
$ cat skills.json
$ ls projects/
React
Go
Python
Java

→ 左：终端持续交互（可输入命令）
→ 右：Bento Grid 技能标签 + 项目卡片
→ 一屏展示全部信息，无需滚动
→ 信息密度高，视觉冲击强

C. Layers — Terminal Overlay on Bento Background

$ whoami
Jamie | Vibecoding since 2025

→ Bento Grid 作为背景层（玻璃模糊）
→ 终端窗口漂浮在背景之上
→ 打字完成后，终端最小化，背景 Bento 清晰显示
→ 叙事性强，像科幻电影 UI

My Recommendation: A (Terminal-First, Scroll-Down)

Pros

  • 传统滚动节奏，招聘方最熟悉
  • Terminal 聚焦注意力，然后自然导向内容
  • 移动端适配最简单
  • 打字动画可以轻松实现

Cons

  • 不如 B/C 视觉新奇
  • 需要滚动才能看到全部内容

A 最稳健且实用——求职站的第一目的是清晰传达信息，A 的叙事节奏最好。B 和 C 可以作为未来迭代加分项。
