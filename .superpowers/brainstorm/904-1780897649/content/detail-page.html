CoBoard

A Kanban collaboration tool that makes cross-department work transparent. Built in one week to demonstrate React ecosystem mastery.

Overview

Small-to-medium dev teams often struggle with cross-department visibility. Frontend doesn't know what backend is blocking on. PM can't see actual progress.

CoBoard solves this with a shared Kanban where each department manages their own board, but anyone can submit requests or flag blockers visible to all. Built around a 4-tier permission model that took more design time than code.

The permission system is the core: 6 operations × 3 roles × 3 card types concentrated into 6 pure functions in src/utils/permissions.ts. No scattered auth checks — every component calls the same deterministic rules.

Highlights

01 — Landing Page Motion Design

Mouse-follow radial gradient orbs using useSpring, scroll parallax cards with useScroll + useTransform, staggered entrance animations with whileInView + delay. Every animation has a purpose — not decorative, but guiding the eye through information hierarchy.

02 — Permission Model as Pure Functions

Instead of scattering role checks across components, all permission logic lives in 6 pure functions. Given a user role and a card type, each function returns what operations are allowed. This made the system testable without a browser and trivially auditable.

// 6 pure functions, zero dependencies
export const canEditCard = (user, card) => {
  if (user.role === 'pm') return true;
  if (user.role === 'leader' && user.dept === card.dept) return true;
  return user.id === card.authorId;
};

03 — Type Safety Without Leaks

User and UserSafe type separation ensures passwords never reach the frontend. TypeScript enforced what code review would have to catch.
